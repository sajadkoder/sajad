BlitzKode

Coding model · Qwen2.5-1.5B · GGUF · llama.cpp

BlitzKode started as a way to understand the full path behind a coding assistant: data, fine-tuning, conversion, inference, and release. I wanted to see how useful a small model could be for focused coding tasks, without treating it like a magic replacement for larger models.

What I built

I fine-tuned Qwen2.5-1.5B on code-instruction data, then converted the result to GGUF so it could run with llama.cpp. The goal was not to beat frontier models; it was to learn how small coding models behave when the whole pipeline is under my control.

Why it matters to me

This project connects directly to my interest in AI developer tools and coding agents. A coding assistant is only useful when it fits into a workflow, responds quickly enough, and can be evaluated against real tasks. BlitzKode helped me think about those trade-offs more clearly.

What I learned

  • Fine-tuning is only one step; data quality and evaluation matter just as much.
  • Small models can be useful for narrow tasks, but they need honest expectations.
  • Packaging and inference details change how usable a model feels.
