Fine-tuned Coding Assistant with Qwen2.5-1.5B

BlitzKode is an AI-powered coding assistant built with a fine-tuned Qwen2.5-1.5B model. It uses llama.cpp for inference and PEFT LoRA for efficient fine-tuning. This document covers the architecture, training pipeline, and inference setup.

Architecture

BlitzKode uses a modular architecture with three main components:

  • Base Model: Qwen2.5-1.5B-Instruct
  • LoRA Adapters: Fine-tuned on code datasets
  • Inference Engine: llama.cpp for efficient CPU inference

Technical Specifications

  • Model: Qwen2.5-1.5B-Instruct
  • Parameters: 1.5 Billion
  • Context Length: 4096 tokens
  • Max New Tokens: 512 tokens
  • Quantization: Q4_K_M (4-bit)
  • LoRA Rank: 16
  • LoRA Alpha: 32

Training Pipeline

The fine-tuning uses PEFT (Parameter Efficient Fine-Tuning) with LoRA adapters. Here's the training script:

# Training BlitzKode with PEFT from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer from peft import LoraConfig, get_peft_model # Load base model model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-1.5B-Instruct") tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-1.5B-Instruct") # Configure LoRA lora_config = LoraConfig( r=16, lora_alpha=32, target_modules=["q_proj", "k_proj", "v_proj"], lora_dropout=0.05, task_type="CAUSAL_LM" ) model = get_peft_model(model, lora_config) # Training arguments training_args = TrainingArguments( output_dir="./blitzkode model", num_train_epochs=3, per_device_train_batch_size=4, learning_rate=2e-4, fp16=True, save_strategy="epoch", save_steps=100, ) trainer = Trainer( model=model, args=training_args, train_dataset=train_dataset, tokenizer=tokenizer, ) trainer.train()

Inference Setup

Run BlitzKode with llama.cpp for efficient CPU inference:

# Run BlitzKode with llama.cpp import llamacpp def generate(query, max_tokens=512, temp=0.7): params = llamacpp.Params() params.n_ctx = 4096 params.n_tokens = max_tokens params.temp = temp model = llamacpp.Model("./blitzkode-q4_k_m.gguf", params) # Build prompt prompt = f"<|im_start|>system\nYou are BlitzKode, an AI coding assistant.<|im_end|>\n<|im_start|>user\n{query}<|im_end|>\n<|im_start|>assistant\n" return model(prompt, max_tokens=max_tokens)

Dataset

Fine-tuned on code数据集 from:

  • Python code samples (50%)
  • JavaScript/TypeScript (20%)
  • Shell scripts (15%)
  • SQL queries (10%)
  • Documentation (5%)

Performance

  • Inference Speed: ~30 tokens/sec on M1 MacBook
  • Memory Usage: ~2GB RAM
  • First Token Latency: ~200ms

Conclusion

BlitzKode demonstrates that a 1.5B parameter model can provide useful coding assistance when fine-tuned properly. The combination of Qwen2.5 base with PEFT LoRA enables efficient fine-tuning on consumer hardware while maintaining reasonable inference speed.

The project is open source and available on GitHub.
