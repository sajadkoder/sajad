SpeakSwap: Translation with Speech I/O in the Browser

Published: September 2025 · Live Demo · GitHub

SpeakSwap began as a college mini-project. I wanted to build a translation tool that felt simple to use: type or speak, get the translation, and hear it spoken back.

The final version runs in the browser, supports voice input, handles fallback translation services, and keeps the interface lightweight.

How it works

The app has two sides: a client (Next.js + React) and a thin server proxy. The client handles the UI, speech recognition, and speech synthesis using the Web Speech API. Translation requests go through a server-side proxy that falls back across three free services (MyMemory, LibreTranslate, Lingva) with rate limiting and response caching.

Speech pipeline

The speech feature uses two browser APIs:

  • SpeechRecognition — converts your voice to text. You select the source language, click the mic, and speak. The browser sends audio to its speech engine and returns transcribed text.
  • SpeechSynthesis — reads the translated text aloud. The app maps each language to an appropriate voice using a locale-to-voice lookup table.

One limitation I ran into: auto-detection works for typed text, but not for voice input. The Web Speech API needs a language code before it starts listening, so the user still has to choose the speaking language.

Architecture highlights

  • Three translation backends with automatic fallback — if one service fails, the next is tried
  • Rate limiting on all API routes to stay within free-tier constraints
  • localStorage for translation history and common phrases (12 languages)
  • Custom hooks: useLocalStorage, useDebounce, useKeyboardShortcut
  • Dark mode with system preference detection via prefers-color-scheme
  • Keyboard shortcuts: Escape clears input, Ctrl+Enter translates

What I learned

SpeakSwap looked simple at first, but the details took work: browser differences in speech recognition, API failures, rate limits, caching, and keeping the UI responsive. It was a good reminder that small apps still need careful edge-case handling.
